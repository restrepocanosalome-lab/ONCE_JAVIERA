# Requerimientos de Software Nivel 2

## Atributos de Calidad (ISO 25010)

La norma ISO/IEC 25010 define ocho atributos principales:

1. Adecuación funcional
2. Eficiencia de desempeño
3. Compatibilidad
4. Usabilidad
5. Confiabilidad
6. Seguridad
7. Mantenibilidad
8. Portabilidad

## Historias de Usuario

Las historias de usuario expresan necesidades desde la perspectiva del usuario.

### Estructura

Como **[usuario]**

Quiero **[funcionalidad]**

Para **[beneficio]**

### Ejemplo

Como estudiante

Quiero consultar el menú del día

Para decidir qué almorzar.

## Método INVEST

- **I** Independiente
- **N** Negociable
- **V** Valiosa
- **E** Estimable
- **S** Pequeña
- **T** Testeable

## Criterios de aceptación

Se escriben con la estructura:

- Dado (Given)
- Cuando (When)
- Entonces (Then)

## Proceso de análisis

1. Identificar stakeholders
2. Aplicar técnicas de elicitación
3. Detectar necesidades
4. Convertir necesidades en requerimientos
5. Clasificar RF y RNF
6. Priorizar atributos de calidad

## Stakeholders principales

- Usuarios
- Clientes
- Patrocinadores
- Entidades regulatorias
- Sistemas externos
